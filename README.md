# Probing Self-Resolving Prediction Markets

**A simulation and empirical study of reference-based resolution.**
Course project for *Algorithm Economics (80470223-0)*, Tsinghua University —
Vilhjalmur Arnar Vilhjalmsson (威廉).

This repository reproduces and empirically stress-tests the mechanism from
**"Self-Resolving Prediction Markets for Unverifiable Outcomes"**
(Srinivasan, Karger & Chen, 2025), which elicits and aggregates beliefs about an
outcome that is *never observed* by paying agents against a well-informed
**reference agent** instead of the ground truth.

> 📄 **Start here:** the written report is [`paper/main.pdf`](paper/main.pdf).
> This README is the map for everything behind it.

---

## TL;DR — what we found

The paper's incentive guarantees are entirely theoretical; the mechanism is never
simulated or run on data. We supply that missing evaluation along two axes.

| | **RQ1 — Incentives (simulation)** | **RQ2 — Robustness (real data)** |
|---|---|---|
| **Question** | When a rational agent is free to lie, when is honesty the best response? | Do reference-based payouts track outcome-based payouts on real markets? |
| **World** | Clean synthetic world; paper's assumptions hold by construction | ~90 resolved Polymarket markets; assumptions broken |
| **Finding** | Profit-maximising deviation γ\* → 1 (truthful) as the reference's informational substitutes *k* grow — exactly truthful by *k* ≈ 64 | Self-resolving ≈ verifiable at a late reference (Pearson ≈ 1.00, payout ratio ≈ 0.95); degrades gracefully as the reference moves earlier |
| **Takeaway** | The paper's Theorems 1–3, realised operationally | The residual gap is **outcome leakage**, concentrated in *ranking*, not payout *levels* |

## Reproduce

### 1. Set up

```bash
cd prediction_market_sim
python3 -m venv .venv && source .venv/bin/activate   # optional
pip install -r requirements.txt                      # numpy, matplotlib, requests
```

### 2. Run the synthetic experiments (RQ1 + supporting)

```bash
cd prediction_market_sim
PYTHONPATH=. python3 examples/quickstart.py          # minimal sanity run

python3 experiments/exp_incentives.py                # RQ1: truthfulness vs. k
python3 experiments/exp_kmin.py                       # analytic k_min bound
python3 experiments/exp_aggregation.py               # exact aggregation
python3 experiments/exp_ground_truth.py              # value of an informed reference
python3 experiments/exp_strategies.py                # mixed/strategic populations
python3 experiments/exp_assumptions.py               # break A1/A2 (rationality, common prior)
```

Figures and CSVs land in `prediction_market_sim/results/`.

### 3. Run the empirical study (RQ2)

```bash
cd prediction_market_sim
python3 empirical/run_empirical.py                   # ~200 top-volume markets
python3 empirical/run_fraction_sweep.py              # relative-time reference sweep
```

Outputs land in `prediction_market_sim/empirical/results/`. Polymarket API responses
are cached under `empirical/cache/` (git-ignored; delete to refresh). See the
[empirical section of the simulator README](prediction_market_sim/README.md#empirical-study-on-real-polymarket-data-empirical)
for all CLI flags.

### 4. Build the paper

```bash
cd paper
latexmk -pdf -shell-escape main.tex                  # needs biber + a TeX distribution
```

`-shell-escape` is required because the template loads `minted`; the bibliography uses
the `biber` backend.

## What the mechanism does (one paragraph)

A binary outcome `Y ∈ {0,1}` is never observed. Agents arrive in sequence; each sees
the running forecast plus a private, conditionally-independent signal, and reports a
probability. The market **terminates at a random time**, and the final (best-informed)
agent becomes the **reference**. Earlier agents are paid the **cross-entropy market
scoring rule** of their report against that reference rather than against the truth.
The key insight: a reference with enough independent signals is a good proxy for the
ground truth, so honesty becomes a (strict) equilibrium — *no outcome required*. RQ1
checks that the incentive actually holds; RQ2 checks that the payouts it produces match
the ones a verifiable market would have paid.
